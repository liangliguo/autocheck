# 论文引用提取改进报告

## 问题描述

在提取论文中的引用参考文献时，原有方法会错误地将数学公式中的符号识别为引用标记。例如：
- `x + r ∈ [0, 1]m` 中的 `[0, 1]` 会被误识别为引用
- `概率值在 [0, 1] 范围内` 会被误识别为引用
- `向量空间 [0, 1]^d` 会被误识别为引用

这些都是数学符号，不应该被当作文献引用处理。

## 解决方案

实现了一个智能的上下文分析函数 `_is_likely_citation_context()`，通过以下规则区分真实引用和数学符号：

### 核心识别规则

#### 排除规则（数学符号特征）

1. **数学符号前导** - 前面有 `∈`, `∉`, `⊂`, `⊆`, `+`, `-`, `×` 等数学符号
2. **数学关键词** - 前面有 `range`, `interval`, `from`, `between`, `within` 等关键词
3. **变量表达式** - 匹配 `value in [...]`, `parameter in [...]` 等模式
4. **中文数学表达** - 匹配 `值在 [...]`, `参数在 [...]`, `在范围 [...]` 等模式
5. **特殊 [0,1] 模式** - 区间 `[0, 1]` 后跟字母或幂次符号（如 `[0,1]m`, `[0,1]^d`）

#### 保留规则（真实引用特征）

1. **句尾或标点** - 后面紧跟句号、逗号、分号等标点符号
2. **较大数字** - 包含大于 20 的数字通常是引用
3. **多个引用** - 包含 3 个以上不连续的数字
4. **引用引导词** - 前面有 `shown in`, `according to`, `see`, `参考` 等引导词
5. **中等数字** - 单个数字大于 5 的情况

## 效果演示

### 数学符号（正确排除）

| 文本 | 结果 |
|------|------|
| `x + r ∈ [0, 1]m` | ❌ 不识别为引用 |
| `概率值在 [0, 1] 范围内` | ❌ 不识别为引用 |
| `向量空间 [0, 1]^d` | ❌ 不识别为引用 |
| `range [0, 100]` | ❌ 不识别为引用 |
| `value in [0,1]` | ❌ 不识别为引用 |

### 真实引用（正确识别）

| 文本 | 结果 |
|------|------|
| `该方法有效[1]。` | ✅ 识别为 `[1]` |
| `研究[5]所示` | ✅ 识别为 `[5]` |
| `根据[10, 11, 12]的研究` | ✅ 识别为 `[10], [11], [12]` |
| `as shown in [23, 24]` | ✅ 识别为 `[23], [24]` |
| `previous work [15-20]` | ✅ 识别为 `[15]...[20]` |

### 混合场景（智能区分）

```
文本: "在区间[0, 1]内，已有研究[5]指出该方法有效。"
结果: 仅识别 [5]，排除 [0, 1] ✅
```

## 使用示例

```python
from autocheck.utils.citations import extract_citation_markers

# 数学公式 - 不会被识别为引用
markers = extract_citation_markers("x ∈ [0, 1]m")
# 结果: []

# 真实引用 - 正确识别
markers = extract_citation_markers("该方法有效[5]。")
# 结果: ['[5]']

# 混合场景 - 智能区分
markers = extract_citation_markers("在区间[0,1]内，研究[10]指出...")
# 结果: ['[10]']
```
